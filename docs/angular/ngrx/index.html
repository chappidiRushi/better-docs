NgRx (State Management)

NgRx is a reactive state management library for Angular apps based on Redux principles â€” a single source of truth, immutable state, unidirectional data flow, and RxJS streams.

  • Store â€” global reactive state container

  • Selectors â€” queries that read state reactively
  • Effects â€” sideâ€‘effects handler (e.g., HTTP)
  • Entity â€” helpers for normalized collections
  • ComponentStore â€” local featureâ€‘level store
  • Router Store â€” sync router state with Store

ðŸ§© Defining Feature Stateâ€‹

A feature isolates a slice of global state.

Example
// counter.reducer.ts
import { createReducer, on } from '@ngrx/store';
import { createAction, props } from '@ngrx/store';

// Actions â€” describe events
export const increment = createAction('[Counter] Increment');
export const decrement = createAction('[Counter] Decrement');
export const set = createAction('[Counter] Set', props<{ value: number }>());

// State model
export interface CounterState {
  value: number;
}

const initialState: CounterState = { value: 0 };

// Reducer â€” pure state update logic
export const counterReducer = createReducer(
  initialState,
  on(increment, state => ({ value: state.value + 1 })),
  on(decrement, state => ({ value: state.value - 1 })),
  on(set, (state, { value }) => ({ value }))
);

Register the feature:

Example
import { provideState } from '@ngrx/store';
import { counterReducer } from './counter.reducer';

providers: [
  provideState('counter', counterReducer)
]

ðŸŽ¯ Actions (What Happened)â€‹

Example

ðŸ§® Reducers (How State Changes)â€‹

Pure functions â†’ no side effects, no async.

Example

  { users: [], loading: false },
  on(loadUsers, s => ({ ...s, loading: true })),
  on(loadUsersSuccess, (s, { users }) => ({ ...s, users, loading: false })),
  on(loadUsersFailure, s => ({ ...s, loading: false }))
);

ðŸ” Selectors (Read Data Reactively)â€‹

Query state in a memoized, performant way.

Example
import { createSelector, createFeatureSelector } from '@ngrx/store';

const selectUsersFeature = createFeatureSelector<any>('users');

export const selectUsers = createSelector(
  selectUsersFeature,
  s => s.users
);

export const selectUsersLoading = createSelector(
  selectUsersFeature,
  s => s.loading
);

Use in a component:

Example
import { Component, inject } from '@angular/core';
import { Store } from '@ngrx/store';
import { selectUsers, selectUsersLoading, loadUsers } from './state';

@Component({ standalone: true, template: `` })
export class UsersComponent {
  private store = inject(Store);

  users$ = this.store.select(selectUsers);
  loading$ = this.store.select(selectUsersLoading);

  ngOnInit() {
    this.store.dispatch(loadUsers());
  }
}

âš¡ Effects (Handle Async & Sideâ€‘Effects)â€‹

Effects listen to actions â†’ trigger async work â†’ dispatch new actions.

Example
import { inject, Injectable } from '@angular/core';
import { Actions, createEffect, ofType } from '@ngrx/effects';
import { switchMap, map, catchError, of } from 'rxjs';
import { HttpClient } from '@angular/common/http';

@Injectable()
export class UsersEffects {
  private actions$ = inject(Actions);
  private http = inject(HttpClient);

  loadUsers$ = createEffect(() =>
    this.actions$.pipe(
      ofType(loadUsers),
      switchMap(() =>
        this.http.get<User[]>(`/api/users`).pipe(
          map(users => loadUsersSuccess({ users })),
          catchError(error => of(loadUsersFailure({ error })))
        )
      )
    )
  );
}

Provide effects:

Example
import { provideEffects } from '@ngrx/effects';

providers: [
  provideEffects(UsersEffects)
];

ðŸ“ Best Practicesâ€‹

  • Keep reducers pure â€” no async / side effects
  • Use selectors everywhere â€” not store.source internals
  • Prefer Entity for lists

  • Use ComponentStore for isolated feature logic
  • Lazyâ€‘load feature state with provideState
  • Avoid giant global stores â€” organize by feature

  • Store + Effects â†’ global crossâ€‘app state, server sync, caching
  • ComponentStore â†’ page/featureâ€‘local reactive state

  • RouterStore â†’ routeâ€‘driven logic
